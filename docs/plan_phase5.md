# MaruMesh Phase 5 구현 계획 (제어 평면 백엔드)

본 계획은 서비스 제공자가 전체 네트워크를 관리하고 모니터링할 수 있는 '중앙 제어 평면(Control Plane)' 백엔드 서버의 초기 구현을 목표로 합니다.

## 1. 주요 목표
- **중앙화된 관리**: 서비스 제공자가 전체 장치 등록 현황, 연결 상태, 사용량 통계를 한눈에 파악.
- **Identity 서비스**: 에이전트의 공개 키 등록 및 장치 인증(CSR 처리) 프로세스 자동화.
- **정책 배포**: 중앙에서 설정한 액세스 정책을 개별 에이전트에게 주기적으로 배포.
- **관리자 API**: 제공자를 위한 관리 기능(장치 차단, 키 재발급 등) 제공.

## 2. 핵심 구성 요소

### [License & Billing Management]
서비스 제공자가 수익을 창출하고 사용을 제어하는 핵심 로직입니다.

- **사용권 검증 API**: 에이전트 기동 시 백엔드가 해당 장치의 유효성(구독 만료 여부 등)을 판단합니다.
- **사용량 통계(Usage Tracker)**: 에이전트가 전송한 데이터 양을 백엔드에서 기록하여 과금 데이터로 활용합니다.
- **원격 차단**: 특정 장치나 사용자의 네트워크 접근을 즉시 정지시킬 수 있는 기능.

### [Backend Server] src/cmd/marumesh-server
- **REST/gRPC API**: 에이전트 및 관리자용 API 서버.
- **Persistence (Database)**: SQLite (MVP용) 또는 PostgreSQL을 사용하여 사용자, 장치, 키 정보를 저장.
- **Stat 수집**: 에이전트들로부터 박동(Heartbeat) 및 메타데이터(트래픽 통계) 수신.

### [Agent Integration] src/internal/control/client.go
- 에이전트가 백엔드 서버와 통신하여 자신의 상태를 보고하고 정책을 받아오는 프로토콜 고도화.

## 3. 작업 순서
1. **백엔드 아키텍처 설계**: DB 스키마 및 API 명세 작성.
2. **에이전트/CLI 통합**: `marumesh-agent`와 `marumesh-ctl`을 `marumesh` 단일 바이너리로 통합.
3. **서버 기초 구현**: 장치 등록 및 상태 관리 API 구축.
4. **대시보드(관리 기능)**: 제공자를 위한 전체 현황 조회 엔드포인트 구현.

## 4. 사용자 검증 항목
- [ ] 백엔드 DB로 선호하시는 기술 스택 (예: 가벼운 SQLite vs 확장성 있는 PostgreSQL)
- [ ] 관리자 화면의 최소 기능 요구사항 (예: 단순 CLI vs 웹 대시보드 기초)
- [ ] 암호화 체계: 모든 정책 데이터의 서명 방식 확정
