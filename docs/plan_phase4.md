# MaruMesh Phase 4 구현 계획 (플랫폼 확장 및 SDK)

마지막 단계인 Phase 4는 완성된 MaruMesh 에이전트 기술을 다양한 환경에 배포하고, 개발자들이 쉽게 사용할 수 있는 도구를 제공하는 데 집중합니다.

## 1. 멀티 플랫폼 빌드 (Cross-Platform)
다양한 OS와 하드웨어 아키텍처에서 MaruMesh를 실행할 수 있도록 빌드 자동화를 구축합니다.

- **대상**: Linux (amd64, arm64), macOS (Intel, M1/M2), Windows.
- **도구**: `Makefile` 또는 `GitHub Actions`를 통한 자동 빌드 파이프라인.

## 2. 관리용 CLI 도구 (CLI Tool)
사용자가 에이전트를 편리하게 제어할 수 있는 CLI(`marumesh-ctl`)를 개발합니다.

- **기능**:
  - `status`: 현재 에이전트 상태 및 노드 정보 출력.
  - `login`: SSO 로그인 트리거.
  - `connect <dev-id>`: 특정 장치와 프록시 연결 수립.
  - `dns <host>`: 특정 호스트의 IP 조회 테스트.

## 3. 타국 언어 및 SDK 지원 (SDK 기초)
다른 프로그램에서 MaruMesh의 기능을 불러다 쓸 수 있는 인터페이스를 제공합니다.

- **Go SDK**: 에이전트의 로컬 API를 감싼 라이브러리.
- **연동 가이드**: Python, Node.js 등에서 로컬 소켓을 통해 통신하는 예제 제공.

---

## 작업 순서
1. **Makefile 작성**: 모든 플랫폼 빌드 스크립트 구축.
2. **CLI 아키텍처 설계**: `src/cmd/marumesh-ctl` 개발 시작.
3. **SDK 명세 확정**: 로컬 API 호출 규격 정리 및 Go 클라이언트 구현.
4. **최종 패키징**: 바이너리 및 기본 설정 파일 묶음 구성.

## 사용자 리뷰 필요
- 배포 시 선호하는 형식 (예: `.deb`, `.zip`, 또는 단일 바이너리 배포).
- SDK에서 가장 우선적으로 필요한 기능 (예: '로그인 자동화' vs '동적 연결 제어').
