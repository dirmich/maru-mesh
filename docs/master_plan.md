# MaruMesh 프로젝트 마스터 플랜

이 문서는 MaruMesh 플랫폼의 전체 아키텍처, 단계별 개발 로드맵 및 현재 상태를 아우르는 상세 통합 계획입니다.

## 1. 프로젝트 비전
MaruMesh는 SSO 인증 기반의 자동 구성 암호화 네트워크를 통해 모든 장치, 서버 및 애플리케이션을 안전하게 연결하는 **보안 커넥티비티 커널**을 목표로 합니다.

## 2. 핵심 아키텍처 계층
1. **제어 평면 (Control Plane)**: ID 관리, SSO 인증, 정책 분배 및 릴레이 좌표 제어.
2. **에이전트 (Agent)**: 개별 장치에서 실행되는 보안/연결 핵심 엔진 (Go 기반 단일 바이너리).
3. **로컬 API (Local API)**: 애플리케이션이 네트워킹 로직을 직접 구현하지 않고 에이전트와 통신하는 인터페이스.
4. **암호화 오버레이 네트워크**: 피어 간 직접 연결(P2P) 및 릴레이 폴백을 지원하는 암호화 통신망.

## 4. 보안 설계 원칙
- **장치 로컬 키 관리**: 개인 키는 절대로 장치를 벗어나지 않습니다.
- **상호 인증 (Mutual Auth)**: 모든 연결은 장치 인증서와 단기 토큰을 필수로 요구합니다.
- **상태 비저장 릴레이 (Stateless Relay)**: 릴레이 서버는 데이터를 복호화할 수 없으며 암호화된 패킷만 전달합니다.
- **최소 권한 원칙**: 모든 액세스는 중앙 정책에 의해 엄격히 제어됩니다.

## 5. 현재 진행 상황
현재 **Phase 1 연장선**에 있으며, 에이전트 내에서 실시간으로 프록시를 제어하고 장치를 탐색하는 로컬 API 기능을 완성하고 품질 안정성 작업을 수행하고 있습니다.
