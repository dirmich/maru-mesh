# MaruMesh 구현 계획 (Phase 1 확장)

이 문서는 MaruMesh MVP 인프라를 기반으로 로컬 API 기능을 확장하고 실제 프록시 세션을 관리하기 위한 상세 계획을 담고 있습니다. 모든 소스 코드는 `src/` 디렉토리에 위치합니다.

## 제안된 변경 사항

### 1. 로컬 API 및 SSO 로그인 (Extension)
에이전트를 통해 원격 장치와 연결할 수 있는 인터페이스와 브라우저 기반 인증 기능을 제공합니다.

- **GET /v1/devices**: 네트워크상의 원격 장치 목록을 반환합니다.
- **POST /v1/proxy/tcp**: 로컬 TCP 프록시 터널을 생성합니다.
- **SSO Login Flow**: 에이전트 로그인 명령 시 브라우저를 열어 SSO 인증을 진행하며, 브라우저 자동 오픈이 안 되는 환경(Linux Server 등)을 위해 콘솔에도 직접 인증 링크를 출력합니다. 로컬 콜백 서버를 통해 인증 결과를 수신합니다.

### 2. 터널 관리 (Tunnel Management)
- `TunnelManager`를 API 핸들러와 연동하여 실시간으로 프록시 서버를 시작하고 중지할 수 있도록 합니다.

## 검증 계획

### 자동화된 테스트
- `go test ./src/internal/api/...`를 통해 새로운 API 엔드포인트의 요청 및 응답 로직을 검증합니다.

### 수동 검증
- Unix Domain Socket을 통해 `curl` 명령어로 API 호출을 테스트합니다.
- 생성된 로컬 프록시 포트(예: 10022)를 통해 실제 통신(SSH 등)이 이루어지는지 확인합니다.

## 사용자 검토 필요 항목
- 프록시 생성 시 사용할 기본 로컬 포트 범위 설정.
- 장치 목록 조회 시 반환할 데이터 스키마 상세 정의.
