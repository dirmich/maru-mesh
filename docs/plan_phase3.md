# MaruMesh Phase 3 구현 계획 (고급 네트워킹 및 NAT 트래버설)

Phase 3는 MaruMesh의 핵심 기술인 **오버레이 네트워킹**을 완성하는 단계입니다. 장치 간의 직접적이고 안전한 통신 환경을 구축합니다.

## 1. NAT 트래버설 및 P2P 연결
복잡한 방화벽과 NAT 환경에서도 장치들이 서로를 찾아 직접 연결할 수 있도록 합니다.

- **WireGuard 통합**: 고성능 암호화 터널링을 위해 WireGuard-Go(User-space) 기술을 사용합니다.
- **ICE/STUN 프로토콜**: `pion/ice` 라이브러리를 활용하여 최적의 네트워크 경로를 탐색합니다.

## 2. 릴레이 폴백 (Relay Fallback)
네트워크 환경이 매우 엄격하여 P2P 연결이 불가능한 경우, 암호화된 패킷을 전달해주는 릴레이 서버를 거쳐 통신합니다.

- **상태 비저장 릴레이**: 에이전트는 중앙에서 지정한 최적의 릴레이 노드를 통해 데이터를 전송합니다.

## 3. 내부 DNS (Internal DNS)
장치 ID나 호스트명을 복잡한 IP 주소 대신 사용할 수 있게 합니다.

- **.maru 도메인**: 에이전트 내부에 경량 DNS 서버를 구축하여 `device-a.maru`와 같은 요청을 해당 장치의 오버레이 IP로 변환합니다.

---

## 작업 순서
1. **네트워킹 핵심 라이브러리 검토**: `wireguard-go`, `pion/ice` 연동 프로토타입.
2. **P2P 탐색 로직 구현**: STUN 서버 연동 및 퍼블릭 IP 발견.
3. **가상 네트워크 인터페이스 설정**: TUN 인터페이스를 통한 패킷 라우팅.
4. **내부 DNS 서버 구축**: UDP 53 포트 리스너 및 매핑 테이블 구현.

## 사용자 리뷰 필요
- **권한 문제**: TUN 인터페이스 생성을 위해 사용자 장치에서 관리자 권한(sudo)이 필요할 수 있습니다. 이를 최소화할 수 있는 User-space 프라이빗 네트워크 방식을 우선 검토할까요?
- **DNS 적용 범위**: 시스템 전체 DNS 설정을 변경할지, 아니면 SDK를 통해서만 작동하게 할지 결정이 필요합니다.
