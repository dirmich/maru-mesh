# MaruMesh Phase 2 구현 계획 (액세스 및 정책 관리)

이 문서는 Phase 1(MVP) 이후 보안성과 관리 편의성을 높이기 위한 Phase 2 구현 계획을 담고 있습니다.

## 1. 중앙 정책 엔진 (Policy Engine)
에이전트가 중앙에서 받은 정책을 로컬에서 실시간으로 해석하여 접근을 제어합니다.

- **정책 구조**: JSON 형식을 사용하여 `Source`, `Destination`, `Action`을 정의합니다.
- **태그 기반 매칭**: 장치에 부여된 태그(예: `tag:robot`, `tag:admin`)를 사용하여 유연한 정책 설정이 가능하게 합니다.

## 2. 권한 제어 (RBAC & Tags)
- 사용자 그룹 및 장치 그룹별로 접근 가능한 서비스(포트)를 제한합니다.
- 'Deny by Default' 원칙을 적용하여 명시적으로 허용된 연결만 승인합니다.

## 3. 감사 로그 (Audit Logging)
- 모든 API 호출 및 프록시 연결 시도(성공/실패)를 구조화된 로그로 남깁니다.
- 추후 중앙 제어 평면으로 로그를 전송하여 보안 관제에 활용합니다.

---

## 작업 순서
1. **정책 데이터 모델 설계**: `src/internal/policy` 모듈 생성.
2. **정책 엔진 구현**: 로컬 정책 파일 로드 및 상시 권한 확인 기능.
3. **API 연동**: `POST /v1/proxy/tcp` 호출 시 정책 엔진의 승인 절차 추가.
4. **감사 로거 구축**: `src/pkg/audit` 생성 및 활동 기록 시작.

## 사용자 리뷰 필요
- 정책 매칭 시 우선순위 규칙 (예: 특정 Deny가 모든 Allow보다 우선하는가?)
- 감사 로그의 로컬 저장 용량 제한 설정.
