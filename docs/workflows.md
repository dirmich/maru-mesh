# MaruMesh 운영 워크플로우 (Workflows)

이 문서는 MaruMesh 플랫폼의 두 핵심 주체인 **사용자(User)**와 **서비스 제공자(Service Provider)**가 시스템을 활용하고 운영하는 상세 절차를 설명합니다.

---

## 👤 1. 사용자(User) 워크플로우

사용자는 MaruMesh를 통해 자신의 장치들을 가상 네트워크로 안전하게 연결합니다.

### 단계 1: 설치 및 시작
1. `Makefile`을 통해 빌드된 `marumesh` 통합 바이너리를 다운로드합니다.
2. 터미널에서 에이전트를 실행합니다.
   ```bash
   ./marumesh run --config settings.json
   ```

### 단계 2: 장치 등록 및 로그인
1. 에이전트가 실행되면 백엔드에 장치 고유 ID와 공개 키를 자동으로 등록합니다.
2. 브라우저가 자동으로 열리며 SSO 로그인(Google 등)을 수행합니다.
3. 로그인이 완료되면 에이전트가 인증 토큰을 획득합니다.

### 단계 3: 승인 대기
- 에이전트는 기동 시 백엔드로부터 **사용 승인(License)**을 확인합니다.
- 승인 전까지는 "대기 중" 상태이며 네트워크 기능이 활성화되지 않습니다.

### 단계 4: 서비스 이용
- 제공자의 승인이 완료되면 에이전트가 `.maru` DNS 서버와 P2P 터널링을 가동합니다.
- 사용자는 다음 명령어로 상태를 확인하고 다른 장치에 연결합니다.
  ```bash
  ./marumesh status
  ./marumesh list
  ```

---

## 🏢 2. 서비스 제공자(Service Provider) 워크플로우

제공자는 전체 네트워크 인프라를 관리하고, 사용자의 권한을 제어하며 과금을 관리합니다.

### 단계 1: 중앙 제어 서버 기동
1. 서버 바이너리를 실행하여 에이전트들의 요청을 받을 준비를 합니다.
   ```bash
   ./marumesh-server
   ```

### 단계 2: 장치 모니터링 및 승인
1. `/api/v1/status` API를 통해 현재 시스템에 접속을 시도한 장치 목록을 확인합니다.
2. 신규 사용자나 결제가 완료된 사용자의 장치에 대해 승인을 수행합니다.
   ```bash
   curl -X POST "http://localhost:8080/api/v1/admin/approve?device_id=장치_ID"
   ```

### 단계 3: 정책 관리
1. 중앙 정책 서버(현재 기초 구현 완료)를 통해 사용자 그룹별 접근 권한(태그 기반 RBAC)을 설정합니다.
2. 에이전트들은 주기적으로 이 정책을 내려받아 적용합니다.

### 단계 4: 사용량 확인 및 과금 (Billing)
1. 백엔드는 개별 에이전트가 30초마다 보고하는 `usage_report` 데이터를 수집합니다.
2. 특정 기간의 누적 트래픽(`usage_bytes`)을 바탕으로 요금을 정산하거나, 초과 사용 시 장치를 차단(`unapprove`)합니다.
