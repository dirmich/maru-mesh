# MaruMesh Phase 6 구현 계획 (웹 대시보드 및 통합 관리)

본 계획은 사용자와 서비스 제공자가 웹 브라우저를 통해 MaruMesh 네트워크를 시각적으로 관리하고 제어할 수 있는 통합 웹 인터페이스 구축을 목표로 합니다.

## 1. 주요 목표
- **백엔드 웹 서버**: `marumesh-server`가 직접 관리 화면(Static Files)을 호스팅.
- **사용자용 그룹 대시보드**: 특정 키(SSO 그룹 키)를 통해 자신들의 에이전트 목록 및 상태 관리.
- **관리자용 통합 대시보드**: 전체 사용 현황, 트래픽 통계 및 글로벌 정책(Rule) 설정.
- **프리미엄 UI/UX**: 다크 모드, 유리 효과(Glassmorphism), 동적 애니메이션이 적용된 현대적인 디자인.

## 2. 웹 아키텍처 및 도메인
- **도메인**: `marumesh.lab.highmaru.com` (에이전트 기본 백엔드 URL로 설정).
- **인증 구조**:
    - 사용자: URL 파라미터 또는 세션 키를 통한 그룹별 대시보드 접근.
    - 관리자: 백엔드에서 설정된 마스터 키를 통한 전체 관리망 접근.

## 3. 구현 내용

### [Backend Server Enhancements]
- **Static File Serving**: `/web` 또는 루트 경로로 대시보드 정적 파일 서빙.
- **Dashboard APIs**:
    - `GET /api/v1/dashboard/group?key=...`: 그룹 내 에이전트 상세 정보.
    - `GET /api/v1/dashboard/admin`: 전체 시스템 통계 및 사용량.
    - `POST /api/v1/dashboard/rules`: 글로벌 방화벽/접근 규칙 설정.

### [Web Interface Design]
- **디자인 컨셉**: 고기능성 네트워크 관리 도구에 걸맞은 Sleek & Premium 디자인.
- **핵심 컴포넌트**:
    - 실시간 에이전트 상태 카드.
    - 트래픽 사용량 그래프 (Chart.js 등 활용).
    - 장치 승인/차단 토글 스위치.

## 4. 작업 순서
1. **대시보드 디자인 및 목업 생성**: `generate_image`를 통한 UI 가이드라인 수립.
2. **백엔드 정적 파일 서빙 구현**: Go의 `http.FileServer` 연동.
3. **사용자용 대시보드 구현**: 그룹 키 기반 필터링 및 에이전트 관리 UI.
4. **관리자용 대시보드 구현**: 전체 통계 및 규칙 설정 UI.
5. **에이전트 설정 업데이트**: 기본 백엔드 주소를 `marumesh.lab.highmaru.com`으로 변경.

## 5. 사용자 리뷰 필요 항목
- [ ] 대시보드 접근 키 생성 방식 (예: SSO 로그인 시 자동 생성 vs 관리자 수동 발급).
- [ ] 관리자 페이지의 추가 보안 요구사항 (예: 2FA 등).
- [ ] 선호하는 차트/그래프 라이브러리 (없을 경우 기본 Chart.js 사용).
