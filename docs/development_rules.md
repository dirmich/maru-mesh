# MaruMesh 개발 및 작업 수행 규칙 (Development Rules)

본 문서는 MaruMesh 프로젝트에 참여하는 모든 개발자(및 AI 에이전트)가 준수해야 할 핵심 작업 규칙을 정의합니다.

## 1. 언어 및 문서화 원칙
- **기본 언어**: 모든 기술 문서, TODO 목록, 워크스루, 소스 코드 내 주요 주석은 **한국어**를 기본으로 사용합니다.
- **문서 동기화**: 코드 변경 시 `docs/todo.md`와 `walkthrough.md`를 즉시 업데이트하여 현재 진행 상태를 투명하게 유지합니다.

## 2. 코드 및 아키텍처 규칙
- **단일 바이너리 유지**: 에이전트와 CLI 기능을 별도의 바이너리로 분리하지 않고, `marumesh` 통합 바이너리 체제를 유지합니다.
- **보안 우선 (Security-First)**: 모든 네트워크 통신은 암호화(ED25519, TLS/ICE)를 전제로 하며, 백엔드의 승인(License) 없이는 핵심 기능이 작동하지 않도록 설계합니다.
- **의존성 관리**: 불필요한 외부 라이브러리 사용을 지양하고, 표준 라이브러리 또는 검증된 오픈소스(pion, cobra 등)를 활용합니다.

## 3. 작업 수행 및 검증 절차
- **빌드 검증 필수**: 모든 코드 수정 후에는 반드시 `make build`를 실행하여 빌드 오류가 없는지 확인합니다.
- **변경 내역 기록**: Git 커밋 메시지는 직관적으로 작성하며(feat:, fix:, docs: 등 사용), 중요 변경 사항은 `setup.md`나 `docs/testlocal.md`에 반영합니다.
- **로컬 테스트 우선**: 새로운 기능 구현 시 `bin/marumesh-server`와 연동하여 로컬 환경에서 기능이 정상 작동하는지 먼저 검증합니다.

## 4. AI 에이전트 특화 지침
- 작업을 시작하기 전 항상 `docs/todo.md`를 읽어 현재 맥락을 파악합니다.
- 사용자의 피드백(LGTM 등)은 `implementation_plan.md`에 기록하여 의사결정 근거를 남깁니다.
- 복잡한 작업은 반드시 단계별 태스크로 나누어 진행 상황을 공유합니다.
