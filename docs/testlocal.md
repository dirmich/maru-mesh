## 1. 전제 조건
- Go 1.26.1 이상 설치됨 (로컬 빌드 시)
- Docker 및 Docker Compose 설치됨 (컨테이너 테스트 시)

---

## 2. 테스트 환경 선택

보유하신 환경에 따라 다음 세 가지 방법 중 하나를 선택하여 테스트를 진행할 수 있습니다.

### 방법 A: 통합 쉘 스크립트 사용 (추천 - 가장 빠름)
Docker 없이 로컬 프로세스로 서버와 2개의 에이전트를 즉시 띄웁니다.

```bash
# 실행
GOOGLE_CLIENT_ID="<google-client-id>" GOOGLE_CLIENT_SECRET="<google-client-secret>" ./test-local.sh
```
- **특징**: **백엔드 통합 인증(Polling)** 방식을 사용하므로 복수의 에이전트를 띄워도 포트 충돌이 발생하지 않습니다.
- **종료**: `Ctrl+C`를 누르면 모든 프로세스가 한꺼번에 안전하게 종료됩니다.

---

## 3. 웹 대시보드 및 SSO 인증 테스트

테스트 환경이 가동되면 브라우저를 통해 마루메쉬 네트워크를 관리할 수 있습니다.

1. **대시보드 접속**: `http://localhost:8080`
2. **에이전트 인증**: 
   - 에이전트 터미널에 출력되는 링크를 눌러 SSO 로그인을 진행합니다.
   - 백엔드 서버(8080)가 인증을 완료하면 에이전트가 이를 감지하여 자동으로 등록 프로세스를 완료합니다.
3. **장치 승인**: 
   - `http://localhost:8080/?user=oldtv.cf@gmail.com` 접속 (ADMIN)
   - 목록의 **Approve** 버튼을 클릭하여 장치를 최종 승인합니다.

---

## 4. 유용한 팁
- **네트워크 청소**: Docker 네트워크 충돌이 발생하면 `docker network prune -f`를 실행하세요.
- **파일 모니터링**: 로컬 테스트 시 `tail -f *.log` 명령어로 실시간 로그를 관찰할 수 있습니다.
